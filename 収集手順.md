# カブスカウト 収集手順書

毎日夕方の自動収集タスクはこの手順書に従うこと。
プロジェクトの場所: `/Users/hiroshi/claude code作業フォルダ/kabu-scout/`

## 目的

翌営業日の朝（寄り付き）に向けた日本株デイトレ用の注目銘柄情報を Web 検索で収集し、
`data.js` を更新する。ユーザーは `index.html` をブラウザで開いて確認する。

## 実行モードの判定（最初に行う）

このタスクは1日2回走る（平日18:00と7:00）。現在時刻で動作を切り替える。

- **現在時刻が12時より前 → 朝の更新モード**（「朝の更新モード」の章に従う）
- **12時以降 → 夕方の収集モード**（以下の「実行手順」に従う）

## 朝の更新モード（7:00の回）

当日の寄り付き直前情報で、**前日夕方に作った同じ targetDate のエントリを更新**する。ゼロから作り直さない。

1. 今日（targetDate）が東京市場の休場日（土日祝）ならスキップして終了。
2. WebSearch/WebFetchで以下を収集する（サブエージェント1〜2体で十分）:
   - **米国市場の確定値**: NYダウ・ナスダック・S&P500の終値と変化率（日本時間朝には確定している）
   - **CME日経225先物**の水準（大阪比でギャップの目安になる）
   - **ドル円**の最新水準
   - 夜間に出た大きなニュース・決算（日本株に影響するもの）
   - 前日プラン銘柄のPTS・ADRの動き（確認できる範囲で）
3. data.js 先頭のエントリ（targetDateが今日のもの）を更新する:
   - market.points の「NYダウ(前日)」「ナスダック(前日)」を確定値に差し替え、「CME日経先物」の point を追加、ドル円を更新
   - market.summary / outlook に朝時点の見方を反映（例:「CME先物は大阪比+300円でギャップアップ含み」）
   - プランは新情報がある銘柄だけ scenario / confidenceScore を調整（米国大幅安なら買いプランのスコアを下げる等）。全面書き換えはしない
   - collectedAt を "前日引け後収集・朝7:00更新" にする
4. 前日夕方のエントリが存在しない場合（夕方の回が動かなかった等）は、夕方の収集モードを一通り実行してから朝の情報を反映する。
5. 構文チェック後、GitHubへpushする（2.5参照）。
6. 「targetDate ○/○分を朝更新（CME先物○○円、米市場反映）」の形式で短く報告する。

## 実行手順（夕方の収集モード）

### 0. 営業日の判定

- 今日が土日または日本の祝日（＝東京市場が休場）だった場合は、市場の値動き情報がないため **収集をスキップして終了** してよい。
- 「翌営業日（targetDate）」は土日・日本の祝日を飛ばして決めること。
  - 例: 金曜夕方に実行 → targetDate は月曜。月曜が祝日なら火曜。

### 1. 情報収集（4カテゴリ）

WebSearch / WebFetch を使い、以下の4カテゴリを調べる。並列のサブエージェント（general-purpose）に分担させると速い。
各カテゴリの検索例と出力フォーマットは「付録: 収集プロンプト」を参照。

1. **市況・地合い** — 本日の日経平均・TOPIX・グロース250、前日の米国市場、ドル円、翌営業日の見通し
2. **決算・IR・適時開示** — 本日引け後発表で翌朝動きそうな銘柄（5〜10銘柄）
3. **ストップ高・出来高急増** — 本日の値動き上位と材料、賑わっているテーマ（5〜10銘柄）
4. **SNS・ブログで話題** — X・note・ブログ・YouTube でトレーダーが言及している銘柄（4〜8銘柄）
   - **Xのポストは Yahoo!リアルタイム検索経由で収集する**（Xと提携していてログイン不要でポストが読める。WebFetchで取得可能なことを確認済み 2026-07）:
     - URL形式: `https://search.yahoo.co.jp/realtime/search?p=キーワード`
     - 検索ワード例: 「明日の注目銘柄」「デイトレ 銘柄」「S高」「決算 サプライズ」、当日賑わった個別銘柄名（例:「SUMCO 株」）
     - 誰のポストか（@ID）を detail に記録する。フォロワー数や実績の裏付けは取れないため、思惑・煽りの可能性に留意した書き方にする。
   - X や Threads へ直接ログインはしない。まとめ記事・note 記事等も引き続き併用する。
   - Threads は検索インデックスがほぼ効かず有用な株情報が取れないことを確認済み（2026-07）。`site:threads.net` 検索を軽く試す程度でよく、収穫ゼロなら無理に載せない。

収集後、掲載する全銘柄の終値を取得する（1.4参照）。

主な情報源: 株探(kabutan.jp)、Yahoo!ファイナンス、みんかぶ、トレーダーズウェブ、松井/楽天等の証券会社レポート、note.com、YouTube。

品質ルール:
- 数値（終値・前日比など）は必ず出典 URL とセットで記録する。確認できなければ「不明」とする。
- 噂・思惑レベルの情報は detail にその旨を明記する。
- 銘柄の証券コード（4桁）は必ず確認する。不明なら空文字。

### 1.4 終値の取得

セクション・プランに載せる**全銘柄**について、当日（直近営業日）の終値と前日比(%)を取得し、
日次エントリの `prices` マップに入れる（コード → {close, change}）。画面では各カードに終値チップとして表示され、クリックで株探チャートに飛ぶ。

- 取得方法: WebFetch で `https://kabutan.jp/stock/?code=コード`（英字入りコード 472A 等もそのまま可）。ダメなら `https://finance.yahoo.co.jp/quote/コード.T` か WebSearch。
- 銘柄数が多いのでサブエージェント1〜2体に一括で任せるとよい。
- 確認できなかった銘柄は close を "不明" とする（画面には表示されない）。

### 1.5 トレードプランの生成（情報の統合）

4カテゴリの収集結果を突き合わせ、翌朝の寄り付きに向けた**参考シナリオ**を3〜5銘柄生成する。

選定ロジック:
- **複数カテゴリで重なった銘柄を優先**する（例: 決算×PTS×SNSの3系統で登場 ＝ 注目度最上位）。basis 欄に何が重なったかを明記する。
- 夜間PTSの動き、地合い（市況・翌日のイベント）、テーマの資金の向きを加味する。
- 買い候補だけでなく、**売り警戒（悪決算の戻り売り等）や「見送り推奨」（思惑株の注意喚起）も含めてよい**。
- 各プランには必ず「入り方の目安」「撤退・無効化ライン」「想定リスク」の3点を書く。**リスクと無効化条件のないプランは出さない。**

自信度スコア（confidenceScore, 100点満点）:
- シナリオの根拠の確かさを 0〜100 で採点する。**「勝率」や「儲かる確率」ではなく、シナリオの根拠がどれだけ裏付けられているかの相対評価**。
- 採点の目安:
  - **70〜85**: 事実ベースの強い材料（決算数値・大型で流動性あり）＋複数情報源で確認＋地合いと整合
  - **50〜69**: 材料は明確だが条件付き（寄り天リスク・小型で値幅荒い・テーマ地合い依存など）
  - **30〜49**: 材料はあるが不確実性が高い（詳細非開示・思惑混じり・地合いと逆行）
  - **0〜29**: 思惑・需給イベントのみで根拠が薄い（見送り推奨クラス）
- **85超は付けない**（翌日の値動きに確実はないため）。
- confidence ラベル（高/中/低）も併記する: 70以上=高、40〜69=中、39以下=低。
- **items 配列は confidenceScore の降順に並べ、rank はその順の連番**とする（画面もスコア順に表示される）。

表現ルール（重要）:
- 「買うべき」「確実」「儲かる」などの断定・助言表現は禁止。「〜が基本シナリオ」「〜なら見送り」のようにシナリオと条件で書く。
- 価格や数値は収集で確認できたものだけを使い、創作しない。目標株価の類は書かない。
- 見出しは「参考シナリオであり投資助言ではない」前提を崩さないこと（画面側に免責表示あり）。

### 2. data.js の更新

`data.js` の `window.KABU_DATA` 配列の **先頭** に以下の形式で1日分を追加する。

```js
{
  date: "YYYY-MM-DD",        // 収集日（今日）
  targetDate: "YYYY-MM-DD",  // 翌営業日
  collectedAt: "18:05",      // 収集時刻（実際の時刻）
  market: {
    summary: "本日の東京市場の総括 2〜3文",
    points: [
      { label: "日経平均", value: "39,850円 (+320)", note: "半導体主導で続伸" },
      // TOPIX / グロース250 / NYダウ(前日) / ナスダック(前日) / ドル円 など
    ],
    outlook: "翌営業日の地合い見通しと注目イベント 3〜4文",
    sources: [{ label: "情報源名", url: "URL" }]
  },
  themes: ["賑わっているテーマ名", "..."],
  prices: {
    // セクション・プラン掲載の全銘柄分。close=直近営業日終値（カンマ区切り文字列）、change=前日比%
    "5018": { close: "2,320", change: "+20.83%" },
    "7453": { close: "3,623", change: "-3.44%" }
  },
  plans: {
    note: "翌朝の全体戦略を2〜3文で（地合い・イベント・物色の向きを踏まえた総評）",
    items: [
      {
        rank: 1,                    // confidenceScore降順の連番
        code: "5018",
        name: "MORESCO",
        side: "buy",                // "buy"=買い方向 / "sell"=売り方向 / "avoid"=見送り推奨
        stance: "大幅GU想定・押し目狙い",  // 戦い方の短いラベル
        confidence: "中",           // 高(70+) / 中(40-69) / 低(-39)
        confidenceScore: 62,        // 0〜100。採点基準は上記。85超は付けない
        basis: "好決算S高 × PTS+20.8% × SNS言及（何が重なったか）",
        scenario: "基本シナリオを2〜3文で",
        entry: "入り方の目安（必須）",
        invalidation: "撤退・無効化ライン（必須）",
        risk: "想定リスク（必須）"
      }
    ]
  },
  sections: [
    { id: "kessan", title: "決算・IR・適時開示", items: [/* 下記 item 形式 */] },
    { id: "ugoki",  title: "ストップ高・出来高急増", items: [] },
    { id: "sns",    title: "SNS・ブログで話題", items: [] }
  ]
}
```

item の形式:

```js
{
  code: "7203",                 // 証券コード4桁。不明なら ""
  name: "トヨタ自動車",
  tag: "好決算",                // 短いラベル（好決算/上方修正/ストップ高/X話題 など）
  direction: "up",              // "up"=買い注目 / "down"=売り警戒 / "watch"=様子見
  headline: "材料を1文で",
  detail: "詳細を2〜3文で。数字は具体的に。噂レベルならその旨明記。",
  sources: [{ label: "株探", url: "https://..." }]
}
```

注意:
- 同じ date のエントリが既にあれば新しい内容で **置き換える**（重複させない）。
- 配列は新しい日付が先頭。**直近10件** を超えた古いエントリは削除する。
- 文字列内の `"` はエスケープする。このMacには node が入っていないため、編集後は必ず以下のコマンドで構文チェックする（エラーが出たら修正してから完了報告する）:

```bash
osascript -l JavaScript -e '
var app = Application.currentApplication(); app.includeStandardAdditions = true;
var src = app.read(Path("/Users/hiroshi/claude code作業フォルダ/kabu-scout/data.js"));
var window = {};
eval(src);
"OK entries:" + window.KABU_DATA.length;
'
```

### 2.5 GitHubへの自動アップロード（朝・夕とも必ず行う）

data.js 更新と構文チェックが済んだら、GitHub Pages（https://rossyrich.github.io/kabu-scout/）に反映するため push する:

```bash
cd "/Users/hiroshi/claude code作業フォルダ/kabu-scout" && git add -A && git -c user.name="RossyRich" -c user.email="uepyon316@gmail.com" commit -m "ブリーフィング更新 YYYY-MM-DD 朝/夕" && git push
```

- コミットメッセージの日付と朝/夕は実際に合わせる。

- Pages への反映は push 後1〜2分。

## 付録: 収集プロンプト（サブエージェント用の要点）

各サブエージェントには「今日の日付」「対象営業日」「JSON のみで返すこと」を必ず伝える。

- 市況: 日経平均/TOPIX/グロース250の終値と前日比、米国市場前日結果、ドル円、翌営業日見通し → market 形式の JSON
- 決算・IR: 「決算速報 site:kabutan.jp」「明日の好悪材料」「業績修正 ○月○日」等で検索 → items 形式
- 値動き: 「ストップ高 ○月○日」「値上がり率ランキング ○月○日」「出来高急増」等で検索 → items + themes 形式
- SNS: Yahoo!リアルタイム検索（`https://search.yahoo.co.jp/realtime/search?p=...`）でXのポストを取得＋「明日の注目銘柄」「デイトレ 注目銘柄 note」「株 注目銘柄 YouTube」等で検索 → items 形式
